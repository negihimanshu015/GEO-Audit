# GEO-Audit

Paste any public URL and get an instant AI citation audit. The tool checks whether your page is structured for AI search engines like ChatGPT, Perplexity, and LLMs to understand, trust, and cite it, then tells you exactly what's missing and gives you a ready-to-use JSON-LD block to implement yourself.

---

## Setup

**Prerequisites:** Python 3.12+, Node 18+, Docker (optional)

### Backend
```bash
cd backend

# Windows
python -m venv venv && venv\Scripts\activate
# Mac/Linux
python -m venv venv && source venv/bin/activate

pip install -r requirements.txt
cp .env.example .env   # fill in GEMINI_API_KEY before running
uvicorn main:app --reload --port 8000
```

**`.env` keys:**
```
GEMINI_API_KEY=...
GEMINI_MODEL=gemini-2.5-flash   
APP_PORT=8000
ALLOWED_ORIGINS=http://localhost:5173
RATE_LIMIT=10 per minute
```

### Frontend

The frontend runs at `http://localhost:5173`. It provides a single-page UI where you paste a URL, trigger the audit, and see the results: extracted metadata, detected schema type, generated JSON-LD, and GEO readiness notes with severity levels.

```bash
cd frontend && npm install
cp .env.example .env.local
npm run dev
```

### Docker

Populate `backend/.env` first, then:
```bash
docker compose up --build
# Backend: localhost:8000 · Frontend: localhost:5173
```

## Architecture

```
URL → [scraper.py] httpx + BeautifulSoup
         → title, meta, headings, images, canonical, author/date, social links
      [schema_rules.py] keyword scoring → schema type hint
      [llm.py] Gemini call → JSON-LD block (may override schema type)
               social links injected into sameAs post-generation
      [audit.py] deterministic rule checks → GEO notes
      → AuditResponse (JSON) → Frontend
```

---

## Design Decisions

**Why rule-based detection + LLM:**
Going fully LLM for schema detection would be slower and non-deterministic. A keyword scorer on URL path and page text is fast and correct for the majority of pages. The LLM then uses that hint to generate the full JSON-LD block, the task it's actually suited for.

**Why GEO notes are deterministic, not LLM-generated:**
Every GEO note is a structural check: presence or absence of a meta tag, heading count, canonical link. These have exact right answers. Using an LLM for if-statements adds cost and latency.

**BeautifulSoup over Playwright:** Server-rendered HTML covers the majority of SEO-relevant pages. Playwright adds a 2s cold start per request.

**Gemini:** Clean async-compatible SDK (`asyncio.to_thread`), and the model is swappable via `GEMINI_MODEL` in `.env` without touching code.

**Edge cases handled:**

| Problem | Fix |
|---------|-----|
| LLM wraps JSON in markdown fences | `_strip_fences()` before `json.loads()` |
| LLM returns invalid/incomplete JSON | Fallback to hardcoded minimal template per schema type |
| LLM hallucinates `sameAs` URLs | `sameAs` never generated by LLM, social links injected after the call |
| Duplicate/garbled headings | `_clean_heading()` detects word and character-level repetition |
| Author/date notes on non-article pages | Gated to `Article`/`BlogPosting`/`NewsArticle` schema types only |

## LLM Usage

**Used for:** JSON-LD generation only, one call per audit. This is the only task requiring knowledge to fill schema fields with page-specific values.

**Not used for:** Schema type detection (keyword scoring), GEO note generation (if-checks), scraping (DOM parser), social link extraction (exact domain pattern matching). All are deterministic and more accurate without an LLM.

---

## Scale-Up (50+ pages)

**Multi-agent pattern:**
```
Orchestrator Agent
  ├── Scraper Agent       : async worker pool; Playwright fallback tier for JS pages
  ├── Schema Agent        : stateless keyword scorer
  ├── LLM Agent           : parallel batched calls, cache
  └── GEO Note Agent      : stateless rule engine
       → Results DB → Frontend
```

**Where LLMs add value vs deterministic logic at scale:**
LLMs are justified only when generating JSON-LD field values. At scale, batch LLM calls by schema type to reuse similar prompts, cache by `url + content-hash` to avoid repeat calls, and set a hard timeout with fallback so one slow API response doesn't block a whole batch. Everything else (detection, notes, scraping) should stay deterministic.

---

## Assumptions & Limitations

- JavaScript-rendered pages (SPAs) return sparse or empty content, no Playwright fallback yet
- Pages behind authentication, paywalls, or bot protection will fail with a scraping error
- `sameAs` is empty if the page doesn't link out to its own social profiles
- 5 schema types supported; unrecognised pages default to `Organization`
- No result caching between runs for the same URL
- Single URL per request
